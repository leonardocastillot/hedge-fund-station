# Market Data Quality Checklist

Use this checklist before trusting strategy conclusions, paper evidence, or UI
rankings. The backend is the source of truth; React may visualize, filter, and
adapt fields for review.

## Source Of Truth By Surface

| Surface | Backend Source | UI Adapter | Source-Of-Truth Fields | Fallback Or Legacy Notes |
| --- | --- | --- | --- | --- |
| Market snapshots | `market_snapshots` SQLite table and `/api/hyperliquid/overview` | `src/services/hyperliquidService.ts` | `symbol`, `price`, `openInterestUsd`, `volume24h`, `fundingRate`, `opportunityScore`, `setupScores`, `primarySetup`, `decisionLabel` | UI labels may group or sort rows, but score and setup fields come from gateway payloads. |
| Alerts | `alerts` SQLite table and `/api/hyperliquid/alerts` | `src/services/hyperliquidService.ts` | `id`, `symbol`, `type`, `severity`, `message`, `value`, `delta`, `createdAt` | Alerts are generated by backend snapshot deltas, not by React. |
| Liquidations | `aggregate_snapshots` plus `/api/liquidations/*` | `src/services/liquidationsService.ts` | `timestamp`, `total_usd`, `longs_usd`, `shorts_usd`, `dominant_side`, `top_markets`, chart metadata | Current liquidation pressure is an estimate derived from gateway market data. UI must keep estimate/source metadata visible. |
| Paper signals | `paper_signals` SQLite table and `/api/hyperliquid/paper/signals` | `src/services/hyperliquidService.ts` | `symbol`, `setupTag`, `direction`, `thesis`, `triggerPlan`, `invalidationPlan`, `entryPrice`, `status`, `createdAt` | Seeding signals is a paper-only action. It must not imply execution permission. |
| Paper trades | `paper_trades` and `paper_trade_reviews` SQLite tables | `src/services/hyperliquidService.ts`, `src/services/strategyService.ts` | `entryPrice`, `exitPrice`, `sizeUsd`, `status`, `realizedPnlUsd`, `review`, `executionQuality`, `decisionLabel` | Legacy portfolio APIs may still display compatibility rows, but strategy promotion evidence comes from gateway paper tables. |

## Session Checks

Before reviewing a strategy:

- `npm run gateway:probe` returns 200 for health, overview, liquidations, paper
  signals, agent runtime, agent runs, and Graphify routes.
- `npm run hf:status` shows expected strategy docs, backend packages, latest
  backtests, validations, and paper candidates.
- `backend/hyperliquid_gateway/data/hyperliquid.db` is local only when debugging;
  canonical heavy runtime state is on the VM under
  `/data/hedge-fund-station/hyperliquid_gateway/data`.
- Market snapshot timestamps are current enough for the reviewed session.
- Liquidation chart metadata includes source, point count, and coverage label.
- Paper trades have review coverage before being used as promotion evidence.

## Red Flags

- A React page computes a strategy trigger that is absent from backend payloads.
- A strategy card appears from a runtime setup row and is mistaken for a
  registered backend strategy.
- `catalogSource` is `strategy-audit-fallback` for normal work after the local
  gateway has supposedly been restarted.
- Liquidation data lacks source metadata or coverage information.
- Paper trades are closed but lack post-trade review notes.
- Local runtime JSON or SQLite data is the only copy of evidence that matters.

## Required Handoff Fields

Any data quality audit should name:

- service URL and probe command used
- artifact or DB path inspected
- endpoint paths inspected
- source-of-truth fields checked
- fallback or legacy fields observed
- stale, missing, null, timestamp, or schema-drift risks
- next validation step
