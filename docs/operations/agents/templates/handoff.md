# Agent Handoff Template

Use this at the end of agent work, especially for recurring or autonomous
improvement tasks.

## Objective

State the objective in one sentence.

## Scope

List the folders, services, strategies, or docs inspected.

## Changes Made

List the concrete changes, grouped by subsystem.

## Files Changed

List the most important files and why they matter.

## Verification

Commands run:

```bash
# command here
```

Result:

- passed
- failed
- skipped, with reason

## Findings

List bugs, risks, stale docs, missing validation, or operational gaps found
during the work.

## Memory Updated

State exactly one:

- added: name the memory file and short reason
- updated: name the memory file and short reason
- promoted: name the canonical doc that now owns the rule
- archived: name the archived/removed item and reason
- removed: name the obsolete item and reason
- intentionally unchanged: explain why this work did not create durable context

## Assumptions

Name any assumptions that affected decisions.

## Next Best Step

Name the single best follow-up task for the next agent.
