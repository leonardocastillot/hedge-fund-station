# Graphify Performance Lite Physics Handoff

## Objective

Keep Graphify opening with the full repo graph by default while making the
custom explorer faster and more comfortable on the current 4,204 node / 6,699
edge dataset.

## Scope

- Inspected and changed the custom explorer generated by
  `backend/hyperliquid_gateway/app.py`.
- Preserved the existing `/memory` iframe integration and source-open contract.
- Added endpoint-level test coverage in `tests/test_graphify_memory_status.py`.
- Did not change `graphify-out/`, Graphify generation, backend strategy logic,
  trading APIs, or public `HyperliquidGraphifyStatus` fields.

## Changes Made

- Added adaptive Graphify explorer profiles:
  - `all-lite` for the full graph: all nodes visible, `barnesHut` physics,
    straight edges, reduced node/edge detail, hover off, shadows off, labels
    off.
  - `focused` for medium filtered views: moderate physics/detail.
  - `neighborhood` for small local views: richer labels, hover, shadows, and
    `forceAtlas2Based` physics.
- Replaced heavyweight full-graph physics with automatic long settling:
  `all-lite` keeps relaxing after stabilization events for a bounded window, so
  the graph keeps organizing itself without requiring a manual `Resume`.
- Changed the `Physics` control into a `Pause` / `Resume` relaxation toggle.
- Debounced search and degree changes so typing and slider movement do not
  rebuild the dataset on every tiny event.
- Added a persistent overlay HUD for visible nodes, visible edges, total nodes,
  community count, active profile, approximate render time, and physics state.
- Kept click selection local: selecting a node still updates styling and the
  inspector without rebuilding the graph.

## Files Changed

- `backend/hyperliquid_gateway/app.py`: custom Graphify explorer HTML, CSS, and
  JS performance profiles, HUD, automatic long settling, debounced refreshes,
  and preserved inspector behavior.
- `tests/test_graphify_memory_status.py`: assertions that the explorer serves
  the adaptive performance profile contract and HUD markup.
- `progress/current.md`: active session state for this task.
- `progress/history.md`: append-only session summary.

## Findings

- The original HUD placement inside `#network` was not durable because
  `vis.Network` owns and clears that container while mounting its canvas. The
  HUD now lives in a graph wrapper as an overlay sibling, so it remains visible
  and does not catch pointer events.
- Full-graph rendering is still intentionally lighter than local neighborhood
  mode, but its layout now keeps settling automatically long enough to recover
  the better organic structure from the previous experience.
- `graphify-out/` remains untouched; regenerate it only when the graph corpus
  itself changes.

## Memory Updated

- intentionally unchanged: this is a product/UX implementation detail, and the
  durable Graphify/Obsidian/file-harness split is already captured in
  `docs/operations/agents/graph-memory-operating-system.md`.

## Assumptions

- "Todo por defecto" means the explorer must still load every node on initial
  open.
- A bounded automatic settling window is acceptable for the full graph because
  it keeps the organic layout behavior without leaving heavy physics running
  forever.
- Staying on the current vis-network explorer is the desired step before any
  future WebGL/Sigma/Cytoscape migration.

## Next Best Step

Profile the same explorer inside packaged Electron on a lower-power machine; if
the graph grows beyond roughly 10k nodes, evaluate a WebGL renderer as a
separate migration task.
